# HDFS Upload & MongoDB Schema/Data Load

**Responsible:** HDFS upload, MongoDB schema & data load
**Project:** Walmart Sales Forecast Big Data Pipeline (COMP/IERG 7095)

---

## Overview

This module implements two storage components of the big data pipeline:

1. **HDFS Upload** -- Upload preprocessed CSV files to HDFS with a structured directory hierarchy, ready for downstream Spark processing.
2. **MongoDB Schema & Data Load** -- Create MongoDB collections with validation rules and analytics-optimized indexes, and load preprocessed data.

### How It Fits in the Pipeline

```
 [Kaggle Dataset]
       |
       v
 +---------------------+
 | Data Preprocessing   |  Jupyter notebook preprocessing
 | (Data_preprocess_    |  creates merged_train.csv,
 |  visualization.ipynb)|  features_cleaned.csv, stores.csv
 +---------------------+
       |
       v
 +---------------------+
 | HDFS Upload (this)   |  preprocessed CSV -> HDFS /preprocessed/
 +---------------------+
       |
       v
 +---------------------+
 | Spark Processing     |  Reads from HDFS /preprocessed/
 |                      |  Joins, Spark SQL, MLlib
 |                      |  Writes to HDFS /enriched/
 +---------------------+
       |
       v
 +---------------------+
 | MongoDB (this)       |  Stores preprocessed data with indexes
 |                      |  for fast analytical queries
 +---------------------+
       |
       v
 +---------------------+
 | Visualization        |  Reads from MongoDB / Spark results
 |                      |  Matplotlib, Seaborn
 +---------------------+
```

## 2. MongoDB Schema & Data Load

### Prerequisites

```bash
pip install pymongo pandas
# MongoDB must be running (localhost:27017 by default, or mongodb:27017 in Docker)
```

### Database Design

**Database:** `walmart_sales`

#### Collections (Preprocessed Data)

| Collection | Documents | Description |
|-----------|-----------|-------------|
| `stores` | 45 | Store metadata: Store (unique), Type (A/B/C), Size |
| `features` | ~8,190 | Cleaned features: (Store, Date) unique, Temperature, Fuel_Price, MarkDowns, CPI, Unemployment, hasMarkDown flags |
| `merged_data` | ~421,570 | Preprocessed merged dataset: train + features + stores + Year/Month/Week |

**Note:** `train_sales` and `test_sales` collections are not created when loading preprocessed data (the merged data contains all training information).

#### Schema Validation

All collections enforce **JSON Schema validation**:
- Required fields checked on insert/update
- Data types enforced: `int`, `double`, `bool`, `date`, `enum`
- MarkDown fields allow `null` (for rows without feature data after join)

#### Index Design

Indexes are optimized for the project's analytical query patterns:

| Query Pattern | Collection | Index |
|--------------|------------|-------|
| Avg sales by store type | `merged_data` | `(Type, Weekly_Sales DESC)` |
| Holiday vs non-holiday sales | `merged_data` | `(IsHoliday, Weekly_Sales DESC)` |
| Markdown impact on sales | `merged_data` | `(hasMarkDown1, Weekly_Sales DESC)` |
| Time-series trend analysis | `merged_data` | `(Date)`, `(Year, Month)` |
| Per-store drill-down | `merged_data` | `(Store, Dept, Date)` unique |
| Top sales ranking | `merged_data` | `(Weekly_Sales DESC)` |

### Usage

#### Option 1: Automatic Upload (via Docker Compose) - Recommended

When using Docker Compose, MongoDB data loading happens automatically:

```bash
cd 7095_proj/

# Start all services (includes automatic data loading)
docker-compose up -d

# Check upload progress
docker logs -f walmart_data_upload

# Verify collections
docker exec -it walmart_tester mongosh --eval "use walmart_sales; show collections; db.merged_data.countDocuments()"
```

#### Option 2: Manual Upload

If not using Docker or need to reload data:

```bash
cd 7095_proj/

# Full pipeline: schema + load preprocessed data + verify
python mongodb_load/mongodb_data_load.py --preprocessed --local-dir ./data/preprocessed

# Custom MongoDB connection
python mongodb_load/mongodb_data_load.py --preprocessed --mongo-uri "mongodb://user:pass@host:27017" --local-dir ./data/preprocessed

# Schema/indexes only (no data)
python mongodb_load/mongodb_schema.py

# Data only (skip schema creation)
python mongodb_load/mongodb_data_load.py --preprocessed --local-dir ./data/preprocessed --skip-schema
```

### Data Load Pipeline Steps

```
Step 1/4: Initialize schema (create collections + validation + indexes)
Step 2/4: Load preprocessed CSVs into collections (stores, features, merged_data)
Step 3/4: Skipped (data already merged in preprocessing)
Step 4/4: Print verification summary with sample analytical queries
```

### Verification (Mongo Shell)

#### Via Docker (recommended when using docker-compose):

```bash
# Access MongoDB shell in container
docker exec -it walmart_mongodb mongosh

# Then run verification commands:
```

#### Via Local MongoDB:

```bash

```

#### Verification Commands:

```javascript
use walmart_sales

// Check document counts
db.stores.countDocuments()        // -> 45
db.features.countDocuments()      // -> ~8190
db.merged_data.countDocuments()   // -> ~421570

// Analytical query: average sales by store type
db.merged_data.aggregate([
  { $group: { _id: "$Type", avg_sales: { $avg: "$Weekly_Sales" } } },
  { $sort: { avg_sales: -1 } }
])

// Analytical query: holiday impact
db.merged_data.aggregate([
  { $group: { _id: "$IsHoliday", avg_sales: { $avg: "$Weekly_Sales" } } }
])

// Analytical query: markdown promotion impact
db.merged_data.aggregate([
  { $group: { _id: "$hasMarkDown1", avg_sales: { $avg: "$Weekly_Sales" }, count: { $sum: 1 } } }
])

// Check indexes
db.merged_data.getIndexes()
```

## Docker Compose Quick Start

The easiest way to get started is using Docker Compose, which automatically starts all services and uploads preprocessed data:

```bash
cd 7095_proj/

# Start all services (MongoDB, HDFS, and automatic data upload)
docker-compose up -d

# Check service status
docker-compose ps

# Monitor data upload progress
docker logs -f walmart_data_upload

# Verify data upload completed
docker exec -it walmart_tester mongosh --eval "use walmart_sales; db.merged_data.countDocuments()"
# Expected output: 421570

# Access services
# - MongoDB: localhost:27017
# - HDFS NameNode UI: http://localhost:9870
# - HDFS DataNode UI: http://localhost:9864

# Stop all services
docker-compose down

# Stop and remove all data volumes
docker-compose down -v
```

### Docker Services

| Service | Container Name | Port | Description |
|---------|---------------|------|-------------|
| MongoDB | `walmart_mongodb` | 27017 | Document database for analytics |
| HDFS NameNode | `namenode` | 9870, 9000 | HDFS master node |
| HDFS DataNode | `datanode` | 9864 | HDFS storage node |
| Data Upload | `walmart_data_upload` | - | Auto-uploads preprocessed data |
| Tester | `walmart_tester` | - | Jupyter/Spark environment for testing |
